## Internal Comms

A set of templates and formatting rules for internal communications in company-standard formats. Covers status reports, leadership updates, third-party updates, company newsletters, FAQs, incident reports, and project updates.

**When to use:** Any request to write internal communications where tone, format, or audience matter.

**Communication type routing:**

| Communication Type | Format |
|-------------------|--------|
| Status report | Structured sections: Summary, Progress, Blockers, Next Steps |
| Leadership update | Exec summary first, detail second; no jargon |
| Team newsletter | Casual, narrative, scannable with headers |
| Incident report | Timeline-based; what happened, impact, resolution, prevention |
| FAQ | Q&A format; plain language; anticipate follow-up questions |
| Project update | RAG status, milestone tracking, risk callouts |

**Key behaviors:**
- Reads the `internal-comms` SKILL.md to load company-specific templates
- Adjusts formality based on audience (team vs. executive vs. all-company)
- Avoids filler phrases, passive voice, and corporate padding
- Keeps summaries genuinely brief — not a condensed version of the full document

## Doc Co-authoring

A structured workflow for co-authoring documentation with the user. Produces better output than single-shot drafting by separating context gathering, structuring, and writing into explicit stages.

**Workflow stages:**
1. **Context gathering** — audience, purpose, scope, constraints, existing materials
2. **Outline** — agree on structure before writing a single word
3. **Draft** — section by section with user review checkpoints
4. **Refinement** — targeted revisions based on feedback
5. **Validation** — confirm the document works for its intended reader

**When to use:** User wants to write documentation, a proposal, a technical spec, a decision doc, or any structured long-form content where iterative collaboration will produce a better result than a single draft.

**When NOT to use:** Short emails, quick summaries, or single-paragraph responses — just write those directly.
